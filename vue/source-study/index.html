# 判断深层次的属性是否存在

业务代码里经常会碰到要判断某个对象的深层次的属性，比如接口返回的对象res：

{
  errno: 0,
  aaaaaaaaaa: {
    bbbbbbbbbb: {
      cccccccccc: true
    }
  }
}

现在要根据res.aaaaaaaaaa.bbbbbbbbbb.cccccccccc的值来区分进行哪项操作，常规的做法是：

if (res.errno === 0) {
  if (res.aaaaaaaaaa && res.aaaaaaaaaa.bbbbbbbbbb && res.aaaaaaaaaa.bbbbbbbbbb.cccccccccc) {
    console.log(res.aaaaaaaaaa.bbbbbbbbbb.cccccccccc)
  }
}
1
2
3
4
5

Vue 源码里学习到的方法：

if (res.errno === 0) {
  let i
  if ((i = res.aaaaaaaaaa) && (i = i.bbbbbbbbbb) && (i = i.cccccccccc)) {
    console.log(i)
  }
}
1
2
3
4
5
6

可以发现，针对对象属性的层次很深时，且每一层属性的key都较长并都需要判断是否存在时，使用不断给i赋值的方式，要方便许多，尤其是在判断存在后获取最后一层属性值时。
